  Section 1: Homepage - Detailed Findings

The homepage is the most critical page - it is the first impression for every visitor arriving via Google, social media, or direct link. Currently, the homepage has received zero of the updates that were applied to the sub-pages, creating a jarring inconsistency.

CRITICALHomepage Is a Different Website from Its Own Sub-Pages
Finding:
The institute sub-pages (JMN, IINR, IIPSR, KGH, IIHMAHS) have all been significantly upgraded - they have real Unsplash hero images, structured content, skip-nav links, WhatsApp, legal footer links, and working CTAs. The homepage has none of these. A visitor arriving on the homepage sees an all-dark, stock-video, ticker-heavy page that looks like a completely different product from the polished sub-pages beneath it. This inconsistency is alarming - it makes the site feel partially finished.
What still needs to be done:
• Apply the same design language used on institute sub-pages to the homepage.
• Replace the dark background with white/off-white.
• Replace the Pixabay video backgrounds with real campus photos (you mentioned you have these - use them now).
• Apply the updated footer structure (with About, Admissions, Careers, NIRF, RTI, Grievance, WhatsApp) to the homepage - currently the homepage footer is still the old bare-bones version.
• Add the skip-nav link to the homepage.
• Update the phone number in the homepage footer from 90000 00000 to the correct number used on sub-pages.
CRITICALHomepage Footer Is the Old Version - Out of Sync
Finding:
The homepage footer only has 3 quick links (Home, About Us #, Admissions #) while every single sub-page footer has the full updated version (About, Admissions, Careers, Sitemap, NIRF, RTI, Grievance, WhatsApp, Trust Reg). The homepage footer links for About Us and Admissions still point to # (dead). The contact number is still the old 90000 00000. This means the two most common trust signals a user checks - footer legal links and phone number - are broken on the most-visited page.
What still needs to be done:
• Replace the homepage footer with the identical footer component used on all sub-pages.
• This is likely a matter of including the same footer component rather than re-coding - check if the homepage is using a different layout or an old footer component version.
• The Trust Reg number should be real, not 'IV-190300XXX (Placeholder)' - update this everywhere once the real number is available.
CRITICALKGH Institute Page Has a Completely Blank Hero Image
Finding:
The Kalyani General Hospital page (institutes/kgh) has an <img> tag with an empty src attribute - no image is loaded at all. This renders as a broken image icon or blank space at the top of the page, which is the first visual a visitor sees. For the foundation's flagship 605-bed hospital, this is a severe presentation failure.
What still needs to be done:
• Add a real photo of the Kalyani General Hospital campus/building to the KGH page hero.

WARNINGScrolling Ticker Unchanged - Still Repeating 4× in a Loop

The marquee ticker reading '? 6 PREMIER INSTITUTIONS ? NMC & INC APPROVED ? 605+ BEDDED HOSPITAL ? GERMANY COLLABORATION' is still present, repeating the same 4 credentials on infinite loop. NEW ISSUEIINR Claims 'First NAAC Accredited Nursing College in West Bengal' - Unverified
Finding:
The IINR page headline states it is 'The First NAAC Accredited Nursing College in West Bengal.' This is an extraordinary claim. NAAC (National Assessment and Accreditation Council) accreditation is a rigorous process - if true, this is a massive differentiator and should be front and center with documentation. If not fully accurate or if the accreditation is pending/in-process, this claim is legally risky.
What still needs to be done:
• If NAAC accreditation is confirmed: add the NAAC certificate scan, the NAAC grade awarded, and the accreditation cycle year prominently on the IINR page.
• If accreditation is in progress: change the language to 'NAAC Accreditation in progress' or 'NAAC Applied'.
• Do not publish 'First NAAC Accredited' without documentary proof - this is a verifiable factual claim that competitors, regulators, and journalists can check.
NEW ISSUEJMN Medical College Claims '98% Average Academic Achievement' - No Source
Finding:
The JMN page states '98% Average Academic Achievement' as a Campus Excellence bullet. This is a marketing metric with no definition, no timeframe, no external verification, and no source. 'Academic achievement' could mean anything. Without specificity it reads as invented. Sophisticated visitors (doctors, academics, other institutions) will immediately distrust it.
What still needs to be done:
• Either remove this metric or replace it with a specific, verifiable claim: e.g., 'MBBS Batch 2023 - 94% students cleared Phase 1 exams on first attempt (Source: JMNMC Academic Office)'.
• Alternatively: 'Board exam pass rate: 96% (2022-23 batch)' with a link to a results page.
• Vague superlatives erode trust more than no data at all.
NEW ISSUEKGH Claims '99.99% Surgical Success Rate' - Extraordinary Claim
Finding:
The Kalyani General Hospital page lists '99.99% Surgical Success Rate' as a campus excellence point. This is an extraordinary medical claim. No publicly known hospital publishes a 99.99% surgical success rate - even the world's top surgical centers do not claim this. Publishing this without source, methodology, timeframe, or external audit makes the entire page look fabricated. It could also attract regulatory scrutiny.
What still needs to be done:
• Remove this claim immediately. It will damage credibility with any medically literate visitor.
• Replace with real, verifiable metrics: number of surgeries performed annually, specific specialty success rates with time period cited, or patient volumes served.
• Example replacement: '12,000+ surgeries performed (2023-24)' or '300+ OPD patients daily' (which is already listed and is a reasonable claim).

  Section 5: Apply Page - Detailed Findings

The /apply page is a new addition and a significant improvement. A working application form was one of the original audit's highest-priority recommendations. Here is a detailed assessment.

FIXEDApplication Form Now Exists with All Core Fields
Finding:
The /apply page now has a proper multi-field application form: Full Name, Date of Birth, Contact Number, Email, Program of Interest (dropdown with 6 programs), Preferred Campus (3 options), 10+2 Percentage, Board Name, PDF Marksheet upload (5MB limit), Declaration checkbox, and a Submit button.
What still needs to be done:
This is a strong foundation. The recommendations below are improvements, not blockers.
NEW ISSUEApplication Form Has No Backend - Submissions Go Nowhere
Finding:
The form exists visually and structurally, but there is no indication of a form backend being connected. There is no action attribute on the form, no visible API endpoint, no Formspree/Netlify form ID, and no confirmation message on submission. If a student fills out and submits this form right now, it is highly likely their application data is lost. This is the most critical gap on the /apply page.
What still needs to be done:
• Connect the form to a backend immediately. Fastest options:
• Formspree (free): add action='https://formspree.io/f/YOURCODE' to the form tag - 5 minutes to set up, sends submissions to email.
• Netlify Forms: if hosted on Netlify, add data-netlify='true' attribute - automatic.
• Google Apps Script: POST to a Google Sheet - free, stores all submissions, easy to monitor.
• Add a visible success state after submission: 'Your application has been received. We will contact you within 3 working days.'
• Send an auto-confirmation email to the applicant's email address.
NEW ISSUEProgram Dropdown Missing M.Sc. Nursing and MHA Programs
Finding:
The Program of Interest dropdown on /apply only offers: MBBS, B.Sc. Nursing, GNM Nursing, B.Pharm, D.Pharm, BHA. Missing programs that are listed on institute pages include: Post Basic B.Sc. Nursing, M.Sc. Nursing, MHA (Master of Hospital Administration), M.Sc. Applied Nutrition. Students interested in these programs cannot accurately complete the form.
What still needs to be done:
• Add all offered programs to the dropdown: Post Basic B.Sc. Nursing, M.Sc. Nursing, MHA, M.Sc. Applied Nutrition.
• Group programs by category using <optgroup> tags: Medical | Nursing | Pharmacy | Administration.
• This makes the dropdown easier to scan for students who know their field but not the exact program name.
NEW ISSUENo NEET Score Field for MBBS Applications
Finding:
MBBS admission in India is governed by NMC and requires NEET-UG qualification. However, the application form has no field for NEET Roll Number or NEET Score - two pieces of information that are mandatory for any MBBS application processing. Without this, MBBS applications submitted through this form cannot be pre-screened or processed.
What still needs to be done:
• Add conditional form fields that appear when 'MBBS' is selected from the Program dropdown:
• NEET Roll Number (text field, numeric)
• NEET Score (numeric)
• NEET Year (dropdown: 2023, 2024, 2025)
• This can be implemented with a simple JavaScript onChange event on the program dropdown.


  Section 6: Design & Visual Consistency

Design is evaluated across the full site - homepage vs sub-pages separately since they are in completely different visual states.

CRITICALTwo Completely Different Design Systems on One Website
Finding:
The homepage uses the original dark aesthetic: black background, ALL CAPS hero, animated ticker, Pixabay videos, minimal footer. Every sub-page uses a cleaner, lighter layout with hero images, structured content blocks, and a full footer. These two design systems have zero visual consistency. What still needs to be done:
• The homepage needs to be rebuilt to match the sub-page design language.
• Key elements to port from sub-pages to homepage: white/light background, real images, static credential strip instead of ticker, full footer component, same nav CTA style.
• The sub-page design is significantly better - the homepage needs to catch up, not the other way around.
PARTIALInstitute Pages Have Good Structure but Identical Layouts
Finding:
All 5 institute pages use an identical template: hero image ? badge ? name ? subheadline ? 4 accordion-style sections (Story, Campus, Admissions, Contacts) ? programs ? accreditations ? back link. The consistency is good for a demo. However, for a production site, key pages like JMN Medical College (the flagship) and KGH (the 605-bed hospital) deserve richer treatment - larger hero, stats strip, photo gallery, more prominent accreditation display.
What still needs to be done:
• Keep the template for smaller institutes (CCNI, ICHFN).

• JMN specifically should have: a dedicated stats row (150 seats, 605-bed attached hospital, Digital Cadaver Lab), a photo gallery section, a faculty section, and an exam results section.
NEW ISSUENo Consistent Brand Color System Across Pages
Finding:
There is no visible consistent accent color being applied across the site. Different pages seem to use different highlight colors with no systematic application. The sub-pages have a cleaner aesthetic but there is no clear primary brand color (e.g., green for healthcare) being applied to: nav CTA, link hover states, section heading accents, badge backgrounds, and button fills. Without a color system, the site looks assembled rather than designed.
What still needs to be done:
• Define one primary brand color and apply it consistently everywhere:
• Recommended: #1D9E75 (forest green) - immediately signals healthcare, trust, growth.
• Apply to: Apply Now button background, active nav link indicator, section heading underline/accent, accreditation badge backgrounds, link hover color, icon fills.
• Secondary: #0F6E56 (darker green) for hover states.
• Neutral: #FAFAF8 for alternating section backgrounds.
• All text links should use the brand green, not default blue.
NEW ISSUENo Typography Consistency - Font Sizes and Weights Vary Arbitrarily
Finding:
Across institute pages, heading sizes, body font sizes, and spacing appear inconsistent. Some pages have very tight spacing between sections; others have more breathing room. There is no apparent typographic scale being enforced. The sub-headline text sizes on institute pages vary. This is a symptom of CSS not being organized into a design token system.
What still needs to be done:
• Define a CSS custom property (design token) system:
• --font-display: 'DM Serif Display', serif (for hero headings)
• --font-body: 'DM Sans', sans-serif (for all body text)
• --text-hero: 3rem; --text-h1: 2.25rem; --text-h2: 1.75rem; --text-h3: 1.25rem; --text-body: 1rem; --text-sm: 0.875rem
• --spacing-section: 5rem (between major sections)
• --spacing-block: 2rem (between content blocks within sections)
• Apply these tokens globally in a _variables.css or in :root {} - never hardcode sizes per-component.


  Section 7: SEO & Technical - Detailed Findings

NEW ISSUEPage Titles Are Generic - All Pages Show 'Nirmala Foundation | Premium Healthcare Education'
Finding:
Every page on the site - homepage, all institute pages, admissions, about, apply - appears to use the same browser tab title: 'Nirmala Foundation | Premium Healthcare Education'. Unique, descriptive page titles are one of the highest-impact SEO factors. Google uses the title tag as the primary ranking signal for page-level queries like 'MBBS college Nadia' or 'nursing college Kalyani'.
What still needs to be done:
• Set unique <title> tags per page:
• Homepage: 'Nirmala Foundation | Healthcare Education | Kalyani, West Bengal'
• JMN: 'JMN Medical College Chakdaha | MBBS | Nirmala Foundation'
• IINR: 'IINR | Best Nursing College Kalyani | B.Sc. Nursing | INC Approved'
• IIPSR: 'IIPSR | B.Pharm D.Pharm | Pharmacy College Kalyani | PCI Approved'
• KGH: 'Kalyani General Hospital | 605-Bed Super Specialty | Nadia'
• Admissions: 'Admissions 2026 | MBBS, Nursing, Pharmacy | Nirmala Foundation'
• Each title should be 50-60 characters, include the primary keyword, and the institution name.
NEW ISSUENo Meta Descriptions on Any Page
Finding:
Meta descriptions (the 160-character snippets shown in Google search results) appear to be absent or generic across all pages. Meta descriptions do not directly affect ranking but they are the primary factor in whether a user clicks on your result vs a competitor's. A well-written meta description for an admissions page can significantly improve click-through rate from NEET/JENPAS-related searches.
What still needs to be done:
• Add unique meta description tags to every page:
• JMN: 'JMN Medical College in Chakdaha, Nadia offers 150-seat MBBS program. NMC approved, attached to 605-bed KGH hospital. NEET-UG admissions open 2026.'
• IINR: 'International Institute of Nursing & Research, Kalyani - NAAC accredited, INC approved. B.Sc. Nursing and Post Basic Nursing programs. Admissions 2026 open.'
• Admissions: 'Apply for MBBS, B.Sc. Nursing, GNM, B.Pharm programs at Nirmala Foundation institutions. Eligibility criteria, scholarships, and Student Credit Card support.'
• Keep all meta descriptions between 140-160 characters.
NEW ISSUENo Open Graph Tags - Poor Social Media Sharing Appearance
Finding:
When any page from this site is shared on WhatsApp, Facebook, or LinkedIn, it will show a blank or generic preview (no image, generic title, no description). In India, WhatsApp is the primary channel through which students share college information with family and friends. Without Open Graph tags, every WhatsApp share of this site looks untrustworthy.
What still needs to be done:
• Add Open Graph meta tags to every page:
• <meta property='og:title' content='JMN Medical College | MBBS | Nirmala Foundation'>
• <meta property='og:description' content='150-seat MBBS college in Chakdaha, Nadia...'>
• <meta property='og:image' content='https://yourdomain.com/images/og/jmn-og.jpg'>
• <meta property='og:url' content='https://yourdomain.com/institutes/jmn'>
• Create dedicated OG images (1200×630px) for each major page - a branded image with the institution name and a key stat.
• Also add Twitter Card tags for completeness.
NEW ISSUENo Schema.org Structured Data on Any Page
Finding:
No JSON-LD structured data is present on any page. Schema.org markup for EducationalOrganization, Course, and Hospital types enables Google rich results - which show star ratings, course details, and address information directly in search results. For local searches like 'nursing college near Kalyani' or 'MBBS college Nadia', structured data can appear in the local knowledge panel.
What still needs to be done:
• Add EducationalOrganization schema to the homepage and About page.
• Add Course schema to each institute page for every program offered.
• Add Hospital schema to the KGH page.
• Add BreadcrumbList schema to all sub-pages.
• Validate with Google's Rich Results Test (search.google.com/test/rich-results) after adding.
• Example for JMN page:
• {"@context":"https://schema.org","@type":"EducationalOrganization","name":"JMN Medical College","url":"...","address":{"@type":"PostalAddress","addressLocality":"Chakdaha","addressRegion":"West Bengal"},"telephone":"+91XXXXXXXXXX"}
IMPROVEImages Have No Alt Text or Generic Alt Text
Finding:
Unsplash images on institute pages use the institute name as alt text (e.g., alt='JMN Medical College') which is better than blank but still not descriptive. Descriptive alt text serves two purposes: accessibility for screen readers, and image SEO (Google Images search). For a healthcare institution, accessibility is also a values signal.
What still needs to be done:
• Rewrite all image alt attributes to be descriptive:
• Not: alt='JMN Medical College'
• Yes: alt='JMN Medical College main academic building, Chakdaha, Nadia, West Bengal'
• Not: alt='International Institute of Nursing & Research'
• Yes: alt='IINR nursing simulation lab with students in clinical training, Kalyani campus'
• When real campus photos are added, write alt text that describes what is actually visible in the photo.
